# Open Source Attribution & Compliance Records

This repository serves as a public archive for documenting instances of license non-compliance and the removal of original authorship metadata within open-source projects.

## Purpose
The integrity of Open Source software relies on the preservation of provenance. Under licenses such as the **GNU General Public License (GPL)**, the requirement to credit original authors and maintain a clear record of changes is a legal mandate, not an optional courtesy.

This log documents the "Direct Injection" and "Commit Laundering" tactics used by project maintainers to strip community-contributed Git metadata, effectively falsifying the historical record of a project's development.

## Methodology
Entries in this archive are verified by comparing:
1. **Original Work:** Publicly available Pull Requests or contributor branches.
2. **Integrated Work:** Commits pushed to an organization's repository.
3. **The Discrepancy:** Evidence of line-by-line logic duplication where the original commit history has been discarded and replaced with maintainer authorship.
